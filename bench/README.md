# smṛti-bench

Cross-platform memory evaluation for advaita-smrti.

## What this measures

A 2×2 experimental design across platforms and memory states:

| | No smṛti | With smṛti |
|---|---|---|
| **Claude Code** | baseline (automated) | treatment (automated) |
| **Gemini API** | — | treatment (automated) |
| **Antigravity** | baseline (manual) | treatment (manual) |

Six prompts targeting: fact recall, decision rationale, task status,
decision coherence, deferred decision recall, and hallucination resistance.

## Quick start

```bash
# Install
pip install "advaita-smrti[mcp]" google-generativeai mcp

# Run automated arms
export ANTHROPIC_API_KEY=...
export GEMINI_API_KEY=...

python bench/run_claude.py --arm baseline --out bench/results/
python bench/run_claude.py --arm smrti    --out bench/results/
python bench/run_gemini.py               --out bench/results/

# Score everything
python bench/score.py bench/results/ --format markdown
```

## Antigravity arm (manual)

1. Copy `bench/results/antigravity_baseline_TEMPLATE.json`
2. Rename: `antigravity_baseline_YYYYMMDDTHHMMSSZ.json`
3. Start a fresh Antigravity session (no prior project context)
4. For the `smrti` arm: add smṛti to `mcp_config.json` first (see below),
   then copy the fixture `.memory/` to your project root
5. Run each prompt from `bench/prompts/battery.txt` verbatim
6. Paste responses into the JSON, commit

**Antigravity mcp_config.json (coding profile):**
```json
{
  "mcpServers": {
    "advaita-smrti": {
      "command": "smrti-mcp",
      "args": ["--profile", "coding"]
    }
  }
}
```

## CI

GitHub Actions runs the automated arms on push to `sutra` and weekly.
Results are committed back to `bench/results/` automatically.

Secrets required:
- `ANTHROPIC_API_KEY`
- `GEMINI_API_KEY`

## Scoring

Responses are scored PASS / FAIL / UNSURE / ERROR against keyword
expectations defined in `score.py`. The hallucination trap (PROMPT_06)
fails if the agent names a specific ORM that is not in memory.

Manual review is always recommended for UNSURE grades.

## Files

```
bench/
  fixtures/
    .memory/
      facts.jsonl       — stable project facts (seeded)
      decisions.jsonl   — architectural decisions with rationale
      tasks.jsonl       — task list with open/closed status
  prompts/
    battery.txt         — 6 prompts with expected answers and traps
  results/
    antigravity_schema.json          — schema for manual Antigravity results
    antigravity_baseline_TEMPLATE.json
    SCORES.md           — generated by CI
    scores.json         — machine-readable scores
  run_claude.py         — Claude Code runner (baseline + smṛti arms)
  run_gemini.py         — Gemini API runner (smṛti arm)
  score.py              — scoring and reporting
  profile_flag_patch.py — proposed --profile flag for smrti-mcp
```
